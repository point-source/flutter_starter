# Project Documentation

This directory is for documentation specific to **your project** -- the app
you are building on top of the `flutter_starter` template.

Template-maintained docs (architecture reference, coding rules, ADRs about the
template's tech stack) live in `docs/template/`.

## What Goes Here

| Directory / File | Purpose |
|---|---|
| `decisions/` | Project-specific ADRs (numbered 100+) -- backend choices, domain conventions, deployment strategies |

You are free to add more subdirectories as your project grows. Common additions:

- `roadmap.md` -- your project's roadmap
- `proposals/` -- your project's design proposals
- `runbooks/` -- operational procedures
- `api/` -- API documentation
- `onboarding.md` -- new developer guide

## Getting Started

See [`decisions/README.md`](decisions/README.md) for the ADR format and
numbering convention.
